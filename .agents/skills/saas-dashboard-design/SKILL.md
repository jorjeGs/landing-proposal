# Goal
Create intuitive, clean, data-dense, and highly functional layouts for SaaS portals and dashboards.

# Instructions
1. Implement a clear navigation layout (e.g., collapsible sidebar navigation, top bar controls).
2. Organise components into grid-based cards to contain charts, lists, metrics, and configurations.
3. Establish a clear layout for Key Performance Indicators (KPIs) above the fold.
4. Ensure data tables are clean, scannable, and support sorting, filtering, and pagination.
5. Apply a consistent color palette to state indicators (e.g., blue for info, yellow for warning, red for error, green for success).

# Input
Dashboard features, data sources, user metrics, and navigation specifications.

# Output
Dashboard layouts, data tables, metrics cards, and visualization frameworks.

# Best Practices
- Prioritize visual clarity over complex layouts; ensure the user's focus goes to actionable insights first.
- Make sure data tables handle overflow content gracefully on smaller screens (e.g., swipe table, toggle columns).
- Keep loading states (skeletons) smooth during dashboard data fetching.
