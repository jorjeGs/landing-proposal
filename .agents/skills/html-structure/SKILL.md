# Goal
Ensure web pages are structured using modern HTML5 semantic elements, promoting SEO, accessibility, and clean document structure.

# Instructions
1. Use HTML5 semantic elements (e.g., <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>) instead of generic <div> tags.
2. Establish a clear heading hierarchy with a single <h1> per page and logically nested headings (<h2> through <h6>).
3. Ensure correct and native interactive elements are used (e.g., <a> for navigation, <button> for actions).
4. Include necessary SEO elements in the <head> (e.g., page titles, meta descriptions, viewport, open graph tags).
5. Apply standard HTML5 attributes like 'lang' on the <html> tag, 'alt' for images, and 'type' for form inputs.

# Input
Structural requirements, page wireframes, mockup designs, or existing unsemantic HTML markup.

# Output
Clean, well-structured, valid HTML5 semantic markup code.

# Best Practices
- Always validate HTML output to ensure conformity with W3C specifications.
- Implement structural SEO by using appropriate tags (like <address>, <time>) where applicable.
- Avoid deep nesting of <div> containers to prevent unnecessary DOM size.
- Make sure text equivalents exist for non-text content.
